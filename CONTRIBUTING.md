Directrices de Contribución
¡Gracias por tu interés en contribuir a nuestro proyecto! Para mantener un entorno colaborativo y eficiente, sigue estas pautas:

Proceso de Pull Request
Para proponer cambios en el proyecto, sigue este flujo de trabajo:

Realiza un Fork del repositorio.

Confirma tus cambios (Commit) en tu propia bifurcación.

Envía un Pull Request. No olvides agregarte al archivo de Authors si corresponde.

Atiende cualquier comentario o retroalimentación antes de que se realice la fusión (merge).

¡Acepta nuestro sincero agradecimiento!

Estándares de Codificación
Para asegurar la calidad del código, nos adherimos a los siguientes estándares:

Python: Seguimos las guías de estilo PEP 8 y las convenciones de nomenclatura de Django.

Lenguajes: Los scripts se escriben para shell bash y el código Python debe ser compatible con la versión 3.6 o superior.

Gestión de Dependencias: Utiliza pipenv para instalar o desinstalar dependencias dentro del shell del proyecto.

Código de Conducta
Al participar, te comprometes a fomentar un ambiente abierto y acogedor. Se espera que todos los colaboradores mantengan una experiencia libre de acoso para todos, independientemente de su identidad, apariencia o nivel de experiencia.

Comportamientos esperados
Usar un lenguaje acogedor e inclusivo.

Ser respetuoso con los diferentes puntos de vista.

Aceptar críticas constructivas con elegancia.

Centrarse en lo que es mejor para la comunidad.
